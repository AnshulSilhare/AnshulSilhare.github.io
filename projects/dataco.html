Machine Learning Case Study

DataCo Supply Chain: AI Delivery Risk Predictor

RoleLead ML Engineer & Developer
DatasetDataCo Supply Chain (180K+ Rows)
DeploymentHugging Face Spaces (Gradio)
Tech StackPython, Scikit-Learn, Pandas, Gradio

Situation

In global supply chains, delay notifications are usually reactive—managers discover a shipment is late only after the SLA has been violated. Reactive management leads to expedited shipping costs, production downtime, and customer dissatisfaction. Proactive visibility requires predicting delay probability before the shipment leaves the warehouse.

Task

My objective was to build and deploy an end-to-end Machine Learning classifier that predicts delivery delinquency at the time of order placement. The solution had to support both real-time order testing via a web interface and bulk processing for hundreds of orders via CSV upload, with model files small enough for standard serverless hosting.

Action

I structured my approach around data preparation, modeling optimization, and front-end interface engineering:

  • Feature Preprocessing: Cleaned 180,519 records. Engineered features for order categories, locations, and shipping modes. One-hot encoded categorical columns, yielding a sparse matrix of 240+ dimensions.
  • Model Size Optimisation: The initial Random Forest model, while accurate, grew to 749MB due to unconstrained tree depth. Applying hyperparameter limits (max_depth=25, min_samples_leaf=3) pruned the forest, shrinking the model size to 22MB while maintaining performance.
  • Matrix Dimension Alignment: The Gradio web form collects only 4 essential parameters from users. The model, however, was trained on 240+ encoded columns. I resolved this mismatch by storing the training feature structure in model.feature_names_in_, generating a zero-filled skeleton matrix on user submit, and mapping the web inputs to their matching encoded indices.
  • Self-Healing Scaler: Programmed a custom scaling routine that uses scaler.feature_names_in_ to dynamically isolate only the continuous variables in the inference data, apply scaling, and re-stitch them with categorical indices, preventing shape exceptions.
  • Interface Deployment: Designed a clean Gradio interface showing risk scores (0–100%) and local feature weights to help coordinators understand *why* an order is flagged.

Result

Quantified System Performance
  • 80% Precision Score: Achieved high accuracy on delinquent delivery flagging, helping managers isolate high-risk shipments prior to routing.
  • 97% Reduction in Model Footprint: Successfully reduced binary size from 749MB to 22MB. This allowed the app to run on Hugging Face's free tier without timeouts.
  • Dual-mode Prediction Backend: Enabled real-time evaluation for single transactions, alongside a batch CSV uploader which evaluates 500+ orders in under 5 seconds.

Reflection

This project proved that in business ML pipelines, model deployability is just as important as accuracy. An unconstrained 749MB model is functionally useless in standard API containers. Investing time in tree pruning and feature engineering is crucial to keep models fast and deployable in production settings.
