SQL Data Warehouse: Medallion Architecture

ArchitectureMedallion (Bronze/Silver/Gold)
SourcesERP (CSV) + CRM (CSV) Systems
ETL CorePure T-SQL Store Procedures
DatabaseMicrosoft SQL Server

Situation

In enterprise settings, sales and customer data reside in disconnected, siloed source platforms—typically an ERP for physical sales and stock, and a CRM for customer contact records. Manually reconciling these flat sheets for analytical reports results in duplicate records, formatting errors, and massive query latencies, preventing stakeholders from making timely business decisions.

Task

My objective was to build a production-grade data warehouse on SQL Server that consolidates siloed ERP and CRM files, processes them through an automated database-level ETL pipeline, and structures them into an optimized Gold analytical layer (Star Schema) for Power BI dashboard reporting.

Action

I designed a 3-layer Medallion Architecture and wrote SQL pipelines to ingest and clean the data:

Data Flow (Medallion Architecture ETL)
CRM & ERP Integration Model
  • Bronze Layer (Raw Ingest): Created staging tables to land CSV source files as-is. Programmed bulk insert schemas to capture raw records, preserving an audit trail of load timestamps.
  • Silver Layer (Cleanse & Standardize): Engineered SQL stored procedures to cleanse the raw data. Wrote logic to handle null values, trim whitespaces, cast data types, and normalize date formats. Deduplicated records using keys to align customer lists across CRM and ERP tables.
  • Gold Layer (Star Schema): Developed the reporting tables, separating transactional business metrics into a central Fact_Sales table connected to optimized dimension tables (Dim_Customers, Dim_Products).
  • Pure T-SQL ETL Pipeline: Wrote stored procedures to automate this data flow sequentially. Designed audit logs tracking pipeline execution run-times and active row counts at each layer.

Result

System Performance & Analytics
  • Production-Grade Architecture: Replaced siloed spreadsheet reviews with a fully automated, Medallion-structured warehouse on SQL Server.
  • ETL Automation: Cleaned, standard database transformations run via simple SQL execution without requiring expensive third-party ETL licensing or pipelines.
  • BI Ready Star Schema: Provided clean, pre-joined dimensional models to power a Power BI sales executive dashboard, reducing dashboard visual load latency to sub-second speeds.
Gold Layer Star Schema Model

Reflection

This project demonstrated that data modeling in the warehouse layer is crucial to ensure analytical scalability. If you feed messy data directly into reporting tools, you create visual lag and calculation errors. Structuring the warehouse into Bronze, Silver, and Gold layers ensures that analytical reports remain fast, clean, and trusted by decision makers.
